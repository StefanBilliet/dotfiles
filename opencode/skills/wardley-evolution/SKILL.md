# Wardley Mapping — Evolution

## Purpose

Assess how evolved each component is.

## Evolution Stages

- Novel
- Custom-built
- Product
- Commodity

## Core Rules

- HANDLE ONE COMPONENT AT A TIME
- ALWAYS require justification
- ALLOW uncertainty
- DO NOT guess

## Interaction

For each component, ask:

- How well understood is this problem?
- Are there existing solutions?
- Are we differentiating here?

Then propose a tentative classification.

## Challenge

- “Why isn’t this a commodity?”
- “Are you rebuilding something that already exists?”
- “What would make this more standardized?”

## Constraints

- NEVER assign without reasoning
- NEVER assume certainty
- NEVER batch all components at once

## Exit Criteria

- Each component has a justified placement (or explicit uncertainty)

## Forbidden

- Strategy
- Value chain changes
