# Wardley Mapping — Value Chain

## Purpose

Identify the components required to fulfill the user need and how they depend on each other.

## Core Rules

- FOCUS on what enables the user need
- BREAK down large components
- IDENTIFY dependencies
- DO NOT discuss evolution yet

## Interaction

Ask:

- What must exist for this user need to be fulfilled?
- What components or capabilities are required?

Then:

- What does each component depend on?
- If this component fails, what breaks?

## Challenge

- “Is this really a separate component?”
- “Is this too broad?”
- “What is this dependent on?”

## Constraints

- Avoid purely technical layers unless relevant
- Avoid vague grouping
- Avoid jumping to solutions

## Exit Criteria

- List of components
- Basic dependency understanding

## Forbidden

- Evolution classification
- Strategic advice
